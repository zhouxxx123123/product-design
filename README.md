## 一、产品概述

### 1.1 战略定位

调研工具不仅是内部效率工具，更是**商业模式的入口**：

```
客户自助梳理（低门槛）→ 发现需专业咨询（转化）→ 咨询师带工具深入（高效率）
```

### 1.2 目标用户

| 维度 | 情况 |
|------|------|
| **规模** | 87人（29个渠道公司 × 每公司3人） |
| **背景** | 销售人员转售前，缺乏专业调研训练 |
| **核心痛点** | 不会提问、不会讲案例、不会回答客户问题 |

---

## 二、核心痛点

1. **专业能力不足** - 销售背景缺乏调研方法论
2. **信息传递失真** - 口头/文字传递导致关键信息衰减
3. **知识无法沉淀** - 每次调研从零开始，案例无法复用
4. **调研质量参差** - 客户觉得不专业，导致丢单

## 四、核心功能详解

### 4.1 功能A：智能提纲生成器

**输入**：客户行业、规模、调研目标、部门清单
**输出**：针对每个部门的苏格拉底式提问清单

**示例**：数字化中心调研提纲（60分钟）

```
1. 现状探索
   "能描述一下现在使用的主要系统吗？"
   → 追问："这些系统之间数据能打通吗？"
   → 追问："哪个系统的使用体验最差？为什么？"

2. 痛点挖掘
   "日常工作中，哪些环节感觉最费时间？"
   → 追问："这个问题影响了什么业务指标？"
   → 追问："如果不解决，半年后会有什么后果？"

3. 元需求提取
   "如果有一个理想的系统，它能帮你做什么？"
   → 追问："这个需求背后的真正目标是什么？"

【预期产出】
- 现有系统清单
- 数据孤岛情况
- 优先级最高的3个痛点
- 元需求提炼
```

### 4.2 功能B：引导提问助手

**引导式互动界面**：屏幕弹出问题，销售对着念，客户回答后弹出下一个问题

```

│                                            │
```

## 五、信息传递机制（混合模式）

销售调研完成后，通过三层信息传递给专家：

| 层级 | 内容 | 专家用途 |
|------|------|----------|
| **第一层：原始记录** | 完整转写文本、带时间戳音频片段 | 核实确认关键问题 |
| **第二层：结构化清单** | 按部门整理的痛点、证据、影响、优先级 | 10分钟掌握细节 |
| **第三层：关键洞察** | 核心痛点TOP3、元需求、潜在风险、建议方案方向 | 1分钟了解全貌 |

**专家工作台**：
- 快速浏览：看第三层（1分钟了解全貌）
- 深入了解：看第二层（10分钟掌握细节）
- 核实确认：听第一层原音（关键问题确认）

## 十、关键洞察总结

1. **这不是"信息收集太慢"的问题** - 而是专业能力不足、信息传递失真、知识无法沉淀的综合问题

2. **引导式交互是核心** - 屏幕弹出问题让销售对着念，降低专业门槛

3. **苏格拉底式提问策略** - 不只是问问题，而是层层追问挖掘真实需求

4. **元需求提取能力** - 从表面需求挖掘背后的真正目标

5. **销售→专家的信息传递** - 报告不给销售，而是给专家完整的信息（原始+结构化+洞察）
