### 追问：信息传递机制设计

**Claude**:
你说报告不给销售，要给专家。但专家没参与调研，怎么知道客户需求？

**现状问题**：销售调研3天，写个简单报告给专家，专家觉得信息不够，还要重新问客户？

**目标**：让专家拿到足够信息，能直接出方案

**选项：**

**A. 结构化需求清单**
```

部门：数字化中心
痛点1：____ 证据：____ 影响：____
痛点2：____ 证据：____ 影响：____
...
```

**B. 访谈原始记录 + AI摘要**
- 销售上传录音/笔记
- AI提取关键对话片段
- 专家可以听原音或看摘要

**C. 问答式记录**
- 每个问题都有记录
- 销售：问了什么
- 客户：回答什么
- AI：提取的洞察

**D. 混合模式**
- 原始记录（给专家听）
- 结构化清单（快速浏览）
- 关键洞察（AI提炼）

## 🎯 MVP功能详细设计

**Claude**:

### 功能A：智能提纲生成器

**输入：**
- 客户基本信息（行业、规模、业务类型）
- 调研目标（数字化升级？流程优化？）
- 调研部门清单

**输出：**
```
调研提纲 - 恩典医疗器械二期

【数字化中心】预计60分钟
目标：了解现有系统、痛点、规划

苏格拉底式提问清单：

1. 现状探索
   "能描述一下现在使用的主要系统吗？"
   → 追问："这些系统之间数据能打通吗？"
   → 追问："哪个系统的使用体验最差？为什么？"

2. 痛点挖掘
   "日常工作中，哪些环节感觉最费时间？"
   → 追问："这个问题影响了什么业务指标？"
   → 追问："如果不解决，半年后会有什么后果？"

3. 元需求提取
   "如果有一个理想的系统，它能帮你做什么？"
   → 追问："这个需求背后的真正目标是什么？"

【预期产出】
- 现有系统清单
- 数据孤岛情况
- 优先级最高的3个痛点
- 元需求：____
```

---
