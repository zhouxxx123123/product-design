## 一、用户认证与权限（9个功能点）

### 1.1 认证模块

**F001 - 账号登录**
- 功能描述：邮箱+密码登录，返回 JWT Access Token + Refresh Token
- 使用角色：ALL
- 前端呈现：✅
- 技术/服务：NestJS JWT、@nestjs/passport

**F002 - Token 刷新**
- 功能描述：Refresh Token 换取新 Access Token，客户端静默完成，用户无感知
- 使用角色：ALL
- 前端呈现：❌（纯后端）
- 技术/服务：JWT Refresh 策略

**F003 - 登出**
- 功能描述：服务端注销 Token，清除客户端存储
- 使用角色：ALL
- 前端呈现：✅

**F004 - 角色权限控制**
- 功能描述：ADMIN / SALES / EXPERT 三角色，前端按角色渲染菜单，后端 Guard 兜底

- 前端呈现：✅
- 技术/服务：RBAC Guard、@Roles()

**F005 - 多租户隔离**
- 功能描述：渠道商数据完全隔离，后端 SQL 层自动过滤，前端无感知
- 使用角色：ALL

### 9.3 实时通信模块

**F072 - 数据加密**

### 9.5 监控模块

### 技术栈分布

- NestJS：核心框架
- PostgreSQL：数据库（含 pgvector、pg_trgm、RLS）

- Socket.io / WebSocket：实时通信
- JWT：认证

- Kimi-k2.5：大语言模型（洞察提取、语义搜索、Copilot）

- 腾讯 COS / MinIO：对象存储

**前端技术**：
- React：Web端
- React Native：移动端（Android）
- Tailwind CSS：样式
- Puppeteer / PDFKit：PDF导出
